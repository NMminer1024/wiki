# Troubleshooting

## 보드가 플래시 도구에 나타나지 않음

- **USB 케이블을 몇 번 다시 꽂아 보십시오** — WebSerial 선택기가 첫 핫플러그 시 기기를 놓칠 수 있으며, 특히 드라이버 설치 직후에 그렇습니다.
- **데이터** USB 케이블을 사용하십시오 (전원 전용이 아닌).
- 다른 USB 포트를 시도하십시오; 일부 허브는 CDC를 통과시키지 않습니다.
- Windows에서는 보드의 브리지 칩에 따라 **CP210x** 또는 **CH340** 드라이버를 설치하십시오.
- `BOOT`을 누른 상태에서 `RESET`을 눌러 다운로드 모드를 강제하십시오.

## Loading에서 멈춤

- WiFi가 **2.4 GHz**인지 확인하십시오 (대부분의 ESP32 칩은 5 GHz를 지원하지 않음).
- 15초 기다리십시오. 연결할 수 없으면 자동으로 AP 모드로 전환됩니다.

## AP `nmap-2.4g`가 보이지 않음

- 보드의 전원을 껐다 켜십시오.
- 구성 버튼을 5초간 길게 누르십시오 ([Buttons](../user-guide/buttons.md) 참조).

## 캡티브 포털이 표시되지만 Save가 작동하지 않음

- 전체 15초 창을 기다리십시오 — 저장소 플러시는 의도적으로 느립니다.
- 페이지가 전혀 응답하지 않으면, 전원을 껐다 켜고 다시 시도하십시오.

## 풀이 "Connecting"에서 멈춤

- 풀 URL과 포트를 확인하십시오.
- `stratum+tcp://` 대신 `stratum+ssl://`을 시도하십시오 (또는 그 반대).
- [대체 풀](../user-guide/pools.md)을 시도하십시오.

## 해시레이트가 사양서보다 훨씬 낮음

- **UI refresh** 설정이 작은 CPU에 부담이 됩니다. 낮춰 보십시오 (폼에서 더 높은 값).
- **Black** 화면 보호기를 사용하여 디스플레이 다시 그리기 비용을 완전히 제거하십시오.
- 전원 공급이 500mA를 유지할 수 있는지 확인하십시오 — 불안정한 USB 포트는 전압 강하를 일으킵니다.

## 모든 셰어가 거부됨

- 풀 난이도가 너무 높게 설정되어 있습니다. NMMiner의 공식 [SoloBTC](https://solobtc.nmminer.com) 풀(또는 공식 대체 풀)로 전환하십시오.
- 지갑 필드를 확인하십시오 — 많은 풀이 잘못된 형식의 지갑 주소를 거부합니다. 유효한 Bitcoin 주소 형식은 [FAQ](./faq)를 참조하십시오.

## NM Monitor에 접근할 수 없음

- 사용자와 마이너가 **동일한 WiFi**에 있는지 확인하십시오.
- 마이너의 Miner 페이지에서 IP를 확인하십시오 — 직접 시도해 보십시오.
- 휴대폰/PC에서 **VPN**을 비활성화하십시오; 일부 VPN은 LAN 트래픽을 차단합니다.
- 라우터가 **AP 격리**를 사용하는 경우, 기기 간 통신이 불가능하므로 해당 모드를 비활성화하십시오.

## Swarm 메뉴에 실제보다 적은 마이너가 표시됨

- **Re-scan**을 클릭하십시오 (또는 페이지 새로고침).
- LAN 스캔은 하나의 /24 서브넷만 스팬합니다. 다른 서브넷의 마이너는 표시되지 않습니다.
- 누락된 마이너가 화면의 **Miner 페이지**에 있는지 확인하십시오 — 검색 스레드는 해당 페이지에서만 실행됩니다.
