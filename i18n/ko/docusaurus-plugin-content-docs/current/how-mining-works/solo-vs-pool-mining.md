# Solo vs Pool Mining — 어떤 것이 나에게 적합할까?

마이닝에 처음 입문하면 **솔로 마이닝**과 **풀 마이닝**이라는 두 용어가 자주 언급되는 것을 들었을 것입니다. 이는 블록 보상을 얻기 위한 두 가지 전략이며, 올바른 것을 선택하는 것이 매우 중요합니다. 쉽게 설명해 드리겠습니다.

---

## 복권 비유

Bitcoin 마이닝을 대략 10분마다 당첨자를 뽑는 거대한 글로벌 복권이라고 상상해 보십시오.

- **상금**: 3.125 BTC (현재 블록 보상) + 거래 수수료.
- **티켓**: 네트워크 난이도를 충족하는 유효한 해시.
- **확률**: 총 네트워크 해시레이트 대비 자신의 해시레이트 비율.

이 그림을 염두에 두고, 두 전략이 어떻게 다른지 살펴보겠습니다.

---

## 솔로 마이닝 — 혼자서 티켓 구매하기

혼자 마이닝합니다. 마이너가 전체 Bitcoin 노드(또는 보상을 분할하지 않는 특별한 솔로 마이닝 풀)와 직접 통신합니다. 독립적으로 블록을 검색합니다.

마이너가 유효한 블록을 찾으면 **보상의 100%를 받습니다**. 공유도, 풀 수수료도 없습니다.

**좋아 보이죠? 문제는 여기 있습니다:**

> 일반적인 NMMiner 해시레이트(~수백 KH/s)로 Bitcoin 네트워크에서 블록을 찾는 평균 시간은 **수백 년** 단위로 측정됩니다. 본질적으로 평생 당첨되지 않을 수도 있는 복권을 하는 것입니다.

### 장점

- 블록을 찾으면 **전체** 블록 보상(3.125+ BTC)을 가집니다.
- 완전한 독립성 — 신뢰할 풀 운영자가 없습니다.
- 풀 수수료가 없습니다.

### 단점

- **극단적인 변동성**: 수년간 마이닝해도 아무것도 얻지 못할 수 있습니다.
- 피드백 루프 없음 — 블록을 찾을 때까지(찾는다면) 설정이 작동하는지 알 수 없습니다.
- 주요 네트워크에서 실용적이려면 막대한 해시레이트가 필요합니다.

### 누가 솔로 마이닝을 해야 할까?

- 정기적으로 블록을 찾을 수 있는 충분한 해시레이트를 가진 대규모 운영.
- 복권으로 취급하고 수익이 0이어도 개의치 않는 취미 사용자.

---

## 풀 마이닝 — 복권 신디케이트 가입하기

마이너 그룹에 가입합니다. 모두 함께 작업하며, 풀 내 **누군가** 블록을 찾으면 각 참가자가 기여한 작업량(**셰어**로 측정)에 따라 보상이 분할됩니다.

블록을 찾기 위해 수백 년을 기다리는 대신, 작지만 **정기적인 지급**을 받습니다 — 때로는 매일, 때로는 매시간.

### 셰어 작동 방식

풀은 네트워크 난이도보다 훨씬 낮은 **풀 난이도**를 설정합니다. 이렇게 하면 NMMiner와 같은 소규모 마이너도 몇 초 또는 몇 분마다 유효한 *셰어*를 제출할 수 있습니다. 이 셰어는 "실제로 검색 중입니다"를 증명합니다. 블록이 발견되면 풀은 모든 사람의 셰어를 집계하여 지급액을 계산합니다.

### 장점

- **예측 가능한 정기 수입** — 결과를 즉시 확인할 수 있습니다.
- **모든 해시레이트**에서 작동합니다 (아무리 작아도).
- 하드웨어가 올바르게 작동하는지 피드백을 받습니다.
- 낮은 위험 — 수익이 시간이 지남에 따라 평활화됩니다.

### 단점

- 풀 수수료 (보통 수익의 0.5%–3%).
- 풀 운영자가 공정하게 지급할 것이라고 신뢰해야 합니다.
- 블록당 수익은 미미합니다 (그러나 쌓입니다).

### 누가 풀 마이닝을 해야 할까?

- **소규모에서 중간 규모 해시레이트를 가진 모든 사람** — 모든 NMMiner 사용자가 포함됩니다.
- 일관되고 예측 가능한 수익을 원하는 사람.
- 결과를 보고 배우고 싶은 초보자.

## 나란히 비교

| | 솔로 마이닝 | 풀 마이닝 |
|---|---|---|
| **느낌** | 복권 티켓 | 정기 급여 |
| **지급 빈도** | 극히 드묾 (아마도 절대 안 됨) | 빈번함 (매일/매시간) |
| **회당 지급 규모** | 막대함 (전체 블록 보상) | 작음 (블록의 본인 지분) |
| **변동성** | 매우 높음 | 낮음 |
| **적합한 해시레이트** | TH/s 이상 | 모든 해시레이트에서 작동 |
| **수수료** | 없음 | 0.5%–3% 풀 수수료 |
| **NMMiner 호환?** | 예 (SoloBTC 풀) | 예 (일반 풀) |

## 진짜 질문: "솔로 마이닝 중인데 왜 풀에 연결하나요?"

대부분의 신규 사용자가 혼란스러워하는 지점입니다. "풀"이라는 단어가 "솔로"의 반대처럼 들리기 때문입니다. 하나씩 풀어보겠습니다.

### 마이닝의 세 가지 방법 — 보상을 바꾸는 것은 두 가지뿐

모든 마이너는 파이프라인 어딘가에 **Bitcoin 노드**가 필요합니다 — Bitcoin 네트워크와 통신하고, 블록체인을 추적하며, 작업을 분배하는 소프트웨어입니다. 문제는: 누가 그 노드를 실행하며, 어떤 종류의 작업을 제공하는가입니다.

가장 DIY 방식부터 가장 손쉬운 방식까지 세 가지 설정이 있습니다:

| 설정 | 노드 실행 주체 | 작업 난이도 | 블록 발견 시 보상 |
|---|---|---|---|
| **진정한 솔로** (자체 노드) | 사용자 | 전체 네트워크 난이도 | 100% 본인 소유 |
| **솔로 풀** (예: SoloBTC) | 풀 운영자 | 전체 네트워크 난이도 | ~98–100% 본인 소유 (소액 수수료) |
| **일반 풀** | 풀 운영자 | 낮음 (셰어 난이도) | 모든 참가자 간 분할 |

:::info

패턴에 주목하십시오: **"솔로"는 보상 모델을 설명하는 것이지, 인프라를 누가 운영하는지가 아닙니다.** 진정한 솔로와 솔로 풀 마이닝 모두에서, 블록을 찾으면 (거의) 전체 블록을 받습니다. 일반 풀에서는 보상이 항상 분할됩니다.

:::

### 진정한 솔로 마이닝 — "순수" 버전

가장 엄밀한 의미에서, 솔로 마이닝은 마이너가 **자체 머신에서 실행 중인 Bitcoin 풀 노드**와 직접 통신하는 것을 의미합니다. 전체 블록체인(~600 GB)을 다운로드하고 24/7 동기화된 상태로 유지하며, 마이너가 이를 대상으로 검색합니다. 블록을 찾으면? 직접 브로드캐스트합니다. 중개인도, 수수료도, 신뢰할 대상도 없습니다.

문제는 인프라 오버헤드입니다. 수백 GB의 여유 디스크 공간이 있는 컴퓨터, 안정적인 상시 연결 인터넷, 노드를 건강하게 유지할 수 있는 기술적 능력이 필요합니다. 대부분의 취미 사용자 — 특히 작은 ESP32 마이너를 실행하는 사람 — 에게 이는 단순히 수고할 가치가 없습니다.

### 솔로 풀 — 노드 골칫거리 없는 솔로 보상

SoloBTC와 같은 풀이 여기에 해당합니다. 이들은 풀 노드를 실행하고 모든 블록체인 배관 작업을 **사용자를 위해** 처리합니다. 그러나 일반 풀과 달리, 작업 난이도를 낮추지 **않고** 블록 보상을 참가자 간에 분할하지 **않습니다**.

대신, 연결된 각 마이너는 **전체 Bitcoin 네트워크 난이도**로 작업을 받습니다. 마이너가 우연히 유효한 블록을 찾으면, 풀이 이를 브로드캐스트하고 보상을 수집한 다음 거의 전액을 사용자 지갑으로 보냅니다 (일반적으로 1–2%의 소액 수수료만 유지).

핵심 통찰: **보상 관점에서 이는 솔로 마이닝입니다.** 인프라 관점에서는 풀 보조 방식입니다. 600 GB 서버를 운영할 필요 없이 솔로 복권 티켓을 얻을 수 있습니다.

### 그렇다면 왜 솔로가 여전히 "복권"인가?

블록을 찾는 난이도는 **풀과 전혀 관련이 없기** 때문입니다 — Bitcoin 네트워크가 설정합니다. 자체 노드를 운영하든 SoloBTC에 연결하든, KH/s급 해시레이트는 여전히 동일한 천문학적 확률에 직면합니다: 평균적으로 블록 간 수백 년.

풀은 인프라 골칫거리를 덜어줍니다. 수학을 바꿀 수는 없습니다.

### NMMiner의 기본값은 무엇인가?

NMMiner는 **SoloBTC**를 기본 풀로 제공합니다 — 즉, 기본적으로 솔로 풀 모드입니다. 플러그를 꽂는 순간 "복권을 즐기는" 재미있고 설정 없는 방법입니다. 그러나 꾸준하고 예측 가능한 수익을 원한다면 일반 풀로 전환하십시오.

:::tip

**NMMiner 사용자용**: Configuration Portal → Pools 탭으로 이동하여 기본 SoloBTC URL을 전통적인 마이닝 풀 URL로 교체하십시오. 단계별 지침은 [Pools](../user-guide/pools.md) 페이지를 참조하십시오.

:::

## 어떤 것을 선택해야 할까?

| 목표 | 권장 사항 |
|---|---|
| "정기적이고 예측 가능한 수익을 보고 싶다" | 일반 풀 |
| "언젠가 전체 블록을 찾는 스릴을 느끼고 싶다" | 솔로 풀 (예: SoloBTC) |
| "대규모 마이닝 팜을 운영하며 정기적으로 블록을 찾는다" | 진정한 솔로 (자체 노드) 또는 솔로 풀 |
| "마이닝 작동 방식을 배우고 있다" | 일반 풀 (즉각적인 셰어 피드백) |

**결론**: 혼란은 보통 이렇게 진행됩니다 — *"잠깐, 솔로 마이닝이면 왜 '풀'에 연결하는 거지?"* 답변은 다음과 같습니다:

- **"솔로"는 보상이 공유되지 않음을 의미합니다.** 블록을 찾으면 그 블록은 당신 것입니다 — 자체 노드를 통해 찾았든, 노드를 대신 실행해 주는 솔로 풀을 통해 찾았든 상관없습니다.
- **"풀"("솔로 풀"에서)은 인프라 계층만을 의미합니다.** 풀이 Bitcoin 노드를 실행하므로 600 GB 하드 드라이브와 24/7 서버가 필요하지 않습니다.

NMMiner 사용자에게 실질적인 교훈: **일반 풀 = 실제로 확인할 수 있는 꾸준한 소액 수익; 솔로 풀 = 인프라 번거로움이 거의 없는 복권 티켓.** 솔로가 "틀린" 것은 아닙니다 — 단지 다른 목표를 위한 것입니다.
