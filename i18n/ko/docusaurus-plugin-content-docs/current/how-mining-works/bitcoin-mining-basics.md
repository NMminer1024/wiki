# Bitcoin Mining Basics

NMMiner가 어떻게 참여하는지 살펴보기 전에, Bitcoin 마이닝에 대한 **한 화면 분량의 멘탈 모델**을 소개합니다.

## 작업

Bitcoin 마이너는 **논스**라는 숫자를 찾기 위해 경쟁합니다 — 이 숫자를 나머지 블록 헤더와 결합하여 SHA-256 해시 함수를 **두 번**(일명 **SHA-256d**) 실행했을 때, 네트워크가 설정한 목표값보다 *작은* 출력을 생성하는 값입니다.

그게 전부입니다. 전체 산업이 하나의 검색 문제입니다:

> *`sha256(sha256(header_with_that_value))`가 네트워크 목표보다 작아지게 하는 32비트 값을 찾으십시오.*

## 난이도, 쉽게 설명하면

- **네트워크 난이도** — *네트워크*가 다음 블록을 획득하는 것을 얼마나 어렵게 만드는지. 오늘날 이는 천문학적으로 높습니다; 합리적인 시간 내에 블록을 찾으려면 ASIC 창고 전체가 필요합니다.
- **풀 난이도** — *풀*이 당신에게 요구하는 검색 난이도. 풀은 소규모 마이너가 **셰어**(실제로 검색하고 있다는 증명)를 생성할 수 있도록 이 값을 훨씬 낮게 설정합니다.
- **셰어** — 풀 난이도를 충족하는(그러나 네트워크 난이도는 아닐 수 있는) 해시 결과. 풀은 셰어에 대해 크레딧을 부여합니다.

NMMiner의 해시레이트는 **수백 KH/s** 범위이므로, 셰어 난이도를 매우 낮게(종종 1 미만의 소수) 설정하는 풀과 통신합니다.

## 마이닝 보상, 쉽게 설명하면

풀이 (연결된 모든 마이너 중에서) 승리 해시를 찾으면, 블록 보상을 획득하고 각 참가자의 셰어 수에 따라 분배합니다. 소규모 마이너는 작은 보상을 받지만, NMMiner가 연결하는 종류의 풀에서는 *지갑으로 직접* 받습니다.

## DigiByte (DGB)도 동일한 방식으로 작동합니다

DGB도 SHA-256d를 사용하므로 수학은 동일합니다 — 네트워크 매개변수와 풀만 다릅니다. NMMiner는 펌웨어 변경 없이 BTC와 DGB를 모두 지원합니다.

## NMMiner의 역할

NMMiner는:

1. 사용자가 구성한 Stratum 풀에 연결합니다 (기본값은 NMMiner 자체 SoloBTC 풀).
2. 풀로부터 **작업** 스트림을 수신합니다.
3. 칩이 허용하는 최대 속도로 논스 공간을 검색합니다.
4. 유효한 셰어를 풀에 다시 전송합니다.
5. 속도, 최신 셰어, 세션/역대 최고 기록을 표시합니다.
