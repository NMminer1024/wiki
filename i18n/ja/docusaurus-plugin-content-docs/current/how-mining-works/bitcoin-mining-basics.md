# Bitcoin マイニングの基本

NMMiner がどのように参加するかに入る前に、Bitcoin マイニングの **1 画面で理解できるメンタルモデル** を説明します。

## 仕事内容

Bitcoin マイナーは、ある数値 — **ノンス** — を見つける競争をします。そのノンスをブロックヘッダーの残りの部分と組み合わせ、SHA-256 ハッシュ関数を **2 回**（別名 **SHA-256d**）通した結果が、ネットワークが設定したターゲット値よりも *小さく* なるものを見つけるのです。

以上です。業界全体が 1 つの探索問題なのです：

> *`sha256(sha256(その値を含むヘッダー))` がネットワークターゲットより小さくなるような 32 ビット値を見つけよ。*

## 難易度を平易に説明すると

- **ネットワーク難易度** — *ネットワーク* が次のブロックを獲得するのをどれだけ難しくしているか。現在、これは天文学的に高く、妥当な時間内にブロックを獲得するには ASIC で満たされた倉庫全体が必要です。
- **プール難易度** — *プール* があなたに求める探索の難しさ。プールはこれをかなり低く設定するため、小規模マイナーでも **シェア**（実際に探索している証拠）を生成できます。
- **シェア** — プール難易度を満たす（ただし必ずしもネットワーク難易度ではない）ハッシュ結果。プールはシェアに対してクレジットを与えます。

NMMiner のハッシュレートは **数百 KH/s** の範囲にあるため、シェア難易度を非常に低く（しばしば 1 未満の端数）設定するプールと通信します。

## マイニング報酬を平易に説明すると

プールが（接続されている全マイナーの中から）勝利ハッシュを見つけると、ブロック報酬を獲得し、各参加者のシェア数に応じて分配します。小規模マイナーは少額の支払いを受け取りますが、NMMiner が接続する種類のプールでは *ウォレットに直接* 支払われます。

## DigiByte (DGB) も同じ仕組みです

DGB も SHA-256d を使用するため、計算は同一です — ネットワークパラメータとプールが異なるだけです。NMMiner はファームウェアを変更することなく BTC と DGB の両方をサポートします。

## NMMiner が行うこと

NMMiner は：

1. 設定された Stratum プールに接続します（デフォルトは NMMiner 独自の SoloBTC プール）。
2. プールから **ジョブ** のストリームを受信します。
3. チップが許す限り高速にノンス空間を探索します。
4. 有効なシェアをプールに送り返します。
5. レート、最新のシェア、セッション / 過去最高のベストを表示します。
