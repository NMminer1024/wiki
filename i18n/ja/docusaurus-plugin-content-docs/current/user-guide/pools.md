# プール

ESP32 のハッシュレートは小さい（数百 KH/s）ため、NMMiner はネットワーク難易度を大きく下回るシェアを受け入れる **低難易度** プールに接続します。

NMMiner は独自のプールに事前設定された状態で出荷されます — **通常は何も変更する必要はありません**。

## 組み込みデフォルト

| スロット       | URL                                              | 地域   |
| ---------- | ------------------------------------------------ | -------- |
| プライマリ    | `stratum+tcp://solobtc.nmminer.com:3333`         | グローバル   |
| フォールバック   | `stratum+tcp://au.solobtc.nmminer.com:3333`      | アジア太平洋 |

どちらも NMMiner の公式 [SoloBTC](https://solobtc.nmminer.com) プールで、数百 KH/s のマイナー向けに設計されています。ダッシュボード: [solobtc.nmminer.com](https://solobtc.nmminer.com)。

プライマリが到達不能になった場合、マイナーは自動的にフォールバックに切り替わります。プライマリが復旧すると、マイナーは約 10 秒以内に切り戻ります（v1.7.05 以降）。

## サポートされるプール URL スキーム

| スキーム               | 意味                                               |
| -------------------- | ----------------------------------------------------- |
| `stratum+tcp://`     | プレーン TCP Stratum v1。                                 |
| `stratum+ssl://`     | TLS ラップド Stratum（プールが SSL を要求する場合に設定）。 |

## サポートされるコイン

NMMiner は現在 **Bitcoin (BTC)** と **DigiByte (DGB)** をサポートしています。両方とも SHA-256d を使用するため、同じハッシュパイプラインが両方に使えます。各コインの既知の正常なエンドポイントについては [プールリスト](../reference/pool-list.md) を参照してください。

## 大手商用プールが使えない理由

NMMiner のハッシュレートは数百 KH/s のオーダーです。高い最小シェア難易度を義務付けるプールは、すべての NMMiner シェアを拒否します。NMMiner の公式 [SoloBTC](https://solobtc.nmminer.com) プールまたはコミュニティのソロスタイルプール — 非常に低いシェア難易度を受け入れるもの — を使用してください。
