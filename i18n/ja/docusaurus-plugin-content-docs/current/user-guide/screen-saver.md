# スクリーンセーバー

設定可能なアイドル時間の後、ディスプレイはスクリーンセーバーモードに入ります。

## モード

| モード    | 効果                                                                                  |
| ------- | --------------------------------------------------------------------------------------- |
| `IMAGE` | NM Monitor 経由でアップロードされたカスタム画像のスライドショー。スムーズなフェードイン / フェードアウト。          |
| `BLACK` | バックライトをオフ（OLED は最小コントラストに）。電力を節約し、**絶対的なピークハッシュレートを得られます**。 |

[NM Monitor](./nm-monitor.md) の **Preferences** セクションからライブでモードを切り替えられます — 再起動不要。

## カスタム画像

NM Monitor の **Screensaver** ページでは、ブラウザから直接通常の PNG / JPG ファイルをアップロードできます。ブラウザが各画像を NMMiner のコンパクトな `.ss` 形式（RGB565 + RLE）に変換し、デバイスの `/ss/` フォルダにプッシュします。

- 画像は **現在の画面解像度** に自動的に合わせられます。
- 各 `.ss` ファイルは最大 **200 KB** です。
- デバイスがディスク容量を管理します — FS がいっぱいになると、最も古いスクリーンセーバーが上書きされます。

アップロードをスクリプト化したい場合は？ [API › スクリーンセーバーアップロード](../api/screensaver-upload.md) を参照。

:::tip
絶対的な最高ハッシュレートを得るには、**Black** モードを選択してください — ディスプレイのリフレッシュはハッシュ計算に使える CPU サイクルを消費します。
:::

## 起動方法

| ボードタイプ      | 起動ジェスチャー                       |
| --------------- | ---------------------------------- |
| ボタンのみ     | 任意のボタンをシングルクリック。           |
| タッチディスプレイ   | 画面のどこかをタップ。           |
